# 知光平台 - 知识获取与分享社区
> 🌟 学习用途项目 | 高并发高可用设计的知识社区APP后端实现

## 项目简介
知光平台是一套面向知识分享场景的社区类APP后端系统，核心支持知识内容发布、点赞/收藏、关注/取关、首页Feed流展示、对象存储直传等核心功能。

项目围绕 **高并发、高可用** 核心目标进行全模块设计，覆盖认证、计数、发布、用户关系、点赞、Feed流、搜索等关键业务场景，是学习分布式系统、高并发架构设计的实践型项目。

**本项目仅作为个人学习用途**

## 技术栈
| 技术/中间件 | 核心用途 |
|------------|----------|
| Java + SpringBoot | 核心开发语言与后端框架 |
| MySQL | 核心业务数据存储 |
| Redis | 计数、缓存、位图判重、会话管理 |
| Kafka | 事件驱动、异步更新、灾难回放、增量数据同步 |
| Caffeine | 本地一级缓存，降低Redis压力 |
| Elasticsearch | 全文检索、前缀联想、权重排序 |

## 核心设计与亮点
### 1. 认证系统 - 安全与性能兼顾
基于 Spring Security 实现 JWT 双令牌认证体系：
- 采用 RS256 非对称加密签名，避免令牌伪造；
- 15分钟短期访问令牌 + 7天刷新令牌分离设计，Redis 维护刷新令牌白名单；
- 支持令牌即时撤销，兼顾高安全性与会话管理性能。

### 2. 计数系统 - 高性能原子计数
针对点赞、收藏、关注等高频计数场景优化：
- Redis 作为底层存储，定制化 SDS 二进制紧凑计数格式，降低内存占用；
- Lua 脚本保证计数原子更新，避免并发问题；
- 内置采样一致性校验 + 自愈重建机制，保证计数数据最终一致。

### 3. 发布系统 - 高效对象存储直传
解决大文件/多媒体发布性能瓶颈：
- 图片、视频、Markdown 文档等内容存储至 OSS 对象存储；
- 后端生成预签名 URL，前端直传存储服务，减少服务端传输开销；
- 渐进式发布流程设计，适配多类型内容发布场景。

### 4. 用户关系系统 - 事件驱动的最终一致性
关注/取关场景的高可用设计：
- 一主多从+事件驱动模型，关注表为主库，粉丝表/计数/列表缓存为伪从；
- 事务内同步写入关注表与 Outbox 表，保证事件不丢失；
- Canal 监听 Outbox 表 binlog，Kafka 异步分发事件，更新下游数据源，解耦同步依赖。

### 5. 点赞系统 - 高并发写优化
应对秒杀级点赞场景的高性能设计：
- 异步写 + 写聚合模式，削峰填谷提升写吞吐；
- Redis 位图结构实现点赞幂等判重，空间效率高；
- 异常场景下基于位图按需重建数据，Kafka 作为“灾难回放”兜底，保证数据不丢失。

### 6. Feed 流 - 三级缓存抗高并发
首页 Feed 流的极致缓存优化：
- 本地 Caffeine → Redis 页面缓存 → Redis 片段缓存三级架构，层层降级；
- 自定义 hotkey 探测，热点内容按层级延长缓存时长，叠加随机抖动避免缓存雪崩；
- 单飞锁(single-flight)机制，防止同一页面并发回源导致的数据库风暴。

### 7. 搜索系统 - 精准检索与低延迟联想
基于 Elasticsearch 的全文检索优化：
- search_after 游标分页替代 from/size，解决深分页性能问题；
- function_score 融合 BM25 相关性与点赞/收藏业务权重，优化检索结果排序；
- completion suggester 实现低延迟前缀联想，提升搜索体验；
- 支持关键词检索、标签过滤，满足多维度内容筛选需求。
